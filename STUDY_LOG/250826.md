## 0) 대충 요약

* 버튼 클릭 시 `isMenuOpen` 상태를 `prev => !prev`로 true → false, false → true 바꾼다
* `isMenuOpen`이 `true`면 **닫기(X)** 아이콘, `false`면 **메뉴(Menu)** 아이콘을 보여준다.
* 모바일 전용 버튼이므로 `md:hidden`으로 **중간 화면 이상에서는 숨김**.
* 접근성: `aria-label`, `aria-expanded`, `aria-controls` 등을 지정한다.
* 아이콘은 `lucide-react`의 `Menu`, `X`를 **반드시 import**.
* Home.jsx에 추가

        </main>

## 1) 최종 코드 (오탈자 수정 + 접근성 보강)

```tsx
// 컴포넌트 상단
import { Menu, X } from "lucide-react"; // ✅ 아이콘 임포트 (대문자 주의)

// 컴포넌트 내부
<button
  type="button"
  onClick={() => setIsMenuOpen((prev) => !prev)} // ✅ 이전 상태 기반 토글
  className="md:hidden p-2 text-foreground z-50" // ⚠️ text-foreground 는 shadcn 테마 변수
  aria-label={isMenuOpen ? "Close menu" : "Open menu"} // ✅ aira → aria
  aria-expanded={isMenuOpen}
  aria-controls="mobile-nav" // 열고 닫는 대상의 id와 연결
>
  {isMenuOpen ? (
    <X size={24} aria-hidden="true" />
  ) : (
    <Menu size={24} aria-hidden="true" />
  )}
</button>
```

> **참고**: `text-foreground`는 shadcn/ui 테마의 CSS 변수입니다. 테마를 쓰지 않는다면 `text-gray-900`(라이트) 또는 `text-white`(다크)처럼 일반 Tailwind 색상 클래스로 바꾸세요.

## 2) 한 줄씩 이해하기

* `onClick={() => setIsMenuOpen((prev) => !prev)}`

  * **prev**는 현재 상태값. `!prev`로 `true/false`를 뒤집어 **토글**.
* `className="md:hidden p-2 text-foreground z-50"`

  * `md:hidden`: **768px 이상** 화면에서는 이 버튼을 숨김(=모바일 전용)
  * `p-2`: 버튼 안쪽 여백 0.5rem
  * `text-foreground`: 글자/아이콘 색. (테마 미사용 시 다른 색상 클래스로 교체)
  * `z-50`: 다른 요소보다 **위에** 보이도록 z-index 크게
* `aria-label`: 스크린 리더용 설명(열림/닫힘 상태에 맞게 문구 변경)
* `aria-expanded`: 현재 메뉴가 **열렸는지(true)**, **닫혔는지(false)** 상태를 기계가 이해하게 표시
* `aria-controls="mobile-nav"`: 이 버튼이 제어하는 **대상 요소의 id**를 알려줌
* `{isMenuOpen ? <X /> : <Menu />}`: 조건부 렌더링. 열림이면 **X**, 아니면 **Menu** 아이콘

## 3) 설치 & 환경 체크리스트

* 아이콘 패키지 설치

  * `npm i lucide-react`
* Tailwind 초기 설정 확인

  * `tailwind.config.{js,ts}`에 `content` 경로가 **src/** 전체를 커버하는지
  * 글로벌 CSS(예: `index.css`)에 `@tailwind base; @tailwind components; @tailwind utilities;` 포함
* shadcn/ui 테마 사용 여부 확인

  * 사용: `text-foreground` 정상 동작
  * 미사용: `text-foreground` → `text-gray-900` 또는 `text-white`로 변경
* Vite alias 설정 (선택)

  * `@`를 `./src`로 매핑했다면 `import { cn } from "@/lib/utils";` 같은 경로가 동작

## 4) 발생한 오류와 해결

* **`Uncaught ReferenceError: X is not defined` / `Menu is not defined`**

  * 원인: 아이콘을 임포트하지 않음, 또는 이름/대소문자 오류
  * 해결: `import { Menu, X } from "lucide-react";` 추가 (정확한 대소문자)


  * 주변 레이아웃의 `z-index` 확인. 버튼에 `z-50` 유지, 상위 요소에 `relative`/`z-10` 등 점검
* **모바일 전용인데 PC에서도 보임**

  * 개발자 도구에서 **화면 폭** 확인. `md:hidden`은 md 이상에서 숨김 → 모바일(작은 화면)에서만 보임이 정상

* [ ] PC 뷰(≥768px)에서 버튼이 **숨겨진다**

## 7) 왜 `prev => !prev` 콜백을 쓰나요?
상태 변경은 바로 안 바뀐다.
React는 상태를 바로 바꾸지 않고 “나중에 한꺼번에” 처리(비동기/배칭)할 수 있어요.

그래서 setIsMenuOpen(!isMenuOpen)처럼 지금 보이는 값으로 계산해버리면,
빠르게 두 번 클릭한 경우 둘 다 같은 옛값을 보고 같은 결과로 설정해 버릴 수 있어요.
→ 결과: 두 번 눌렀는데 한 번만 바뀐 것처럼 보임 

prev => !prev가 해결하는 방법

setIsMenuOpen(prev => !prev)는 **그 순간의 “진짜 최신 값(prev)”**을 React가 넣어줘요.
첫 번째 클릭: false → true
두 번째 클릭(바로 이어서): 이때 prev는 이미 true → true → false
→ 결과: 두 번 누르면 진짜 두 번 뒤집힘 ✅

초간단 비유

!isMenuOpen: “지금 내가 보고 있는 값의 반대로 바꿔줘!”
(내가 본 값이 오래된 값일 수도 있음)

prev => !prev: “그 순간의 실제 최신 값을 받아서 반대로 바꿔줘!”
(항상 최신 상태 기준으로 정확히 뒤집음)

요약: React가 상태를 나중에 묶어서 처리할 수 있어서 값이 **잠깐 ‘옛날 값’**일 수 있어요.
