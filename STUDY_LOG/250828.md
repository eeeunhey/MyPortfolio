# 📘 SkillsSection 카테고리 필터링 기능 학습 로그

## 📌 기능 개요
`SkillsSection` 컴포넌트에 **카테고리별 필터링 기능**을 추가했다.  
버튼(`all`, `frontend`, `backend`, `tools`)을 클릭하면 해당 카테고리의 기술만 화면에 표시된다.  
React의 `useState`, 배열 메서드 `filter`, `map`을 활용하였다.

---

## 🔹 1) 카테고리 배열 정의
```js
const categories = ["all", "frontend", "backend", "tools"];
```
- 버튼에 표시할 카테고리 목록 정의  
- `"all"`은 전체 보기용, 나머지는 `skills` 배열의 `category`와 매칭됨  
- 개선 가능성: 나중에는 `skills`에서 `category`를 추출해 자동으로 생성할 수 있음  

---

## 🔹 2) 상태(state) 관리
```js
const [activeCategory, setActiveCategory] = useState("all");
```
- `activeCategory`: 현재 선택된 카테고리 값 저장  
- `setActiveCategory`: 버튼 클릭 시 상태 갱신  
- 기본값 `"all"` → 처음엔 전체 스킬 표시  

**배운 점**  
React에서 상태값이 UI와 연결되면, 이벤트 발생 시 자동으로 리렌더링되어 화면이 갱신된다.

---

## 🔹 3) 필터링 로직
```js
const filterSkills = skills.filter(
  (skill) => activeCategory === "all" || skill.category === activeCategory
);
```
- `activeCategory === "all"` → 전체 스킬 출력  
- 특정 카테고리 선택 → 해당하는 `skill.category`만 필터링  

**배운 점**  
배열의 `filter` 메서드와 상태를 결합하면, 사용자가 선택한 조건에 맞춰 데이터를 동적으로 보여줄 수 있다.

---

## 🔹 4) 카테고리 버튼 UI
```jsx
<div className="flex flex-wrap justify-center gap-4 mb-12">
  {categories.map((category, key) => (
    <button
      key={key}
      onClick={() => setActiveCategory(category)}
      className={cn(
        "px-5 py-2 rounded-full transition-colors duration-300 capitalize",
        activeCategory === category
          ? "bg-primary text-primary-foreground"
          : "bg-secondary/70 text-foreground hover:bg-secondary"
      )}
    >
      {category}
    </button>
  ))}
</div>
```

### 동작 원리
1. `categories.map()`을 사용하여 버튼 반복 렌더링  
2. 클릭 시 `setActiveCategory(category)` 실행 → 상태 변경  
3. 조건부 클래스 적용  
   - 선택된 버튼: `bg-primary` + 강조 색상  
   - 비활성 버튼: 흐린색(`bg-secondary/70`), hover 시 색상 변경
