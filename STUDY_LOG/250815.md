# 유성우 기능 구현 기록 (StarBackground)

## 1) `generateMeteors` 함수 생성

### 목적

-   한 번 호출로 **화면에 뿌릴 유성(메테오)들의 초기 상태**를 모아
    생성하고 `state`에 저장한다.

### 구성 요소

-   `const numberOfMeteors = 4;`
    -   이번 호출에서 생성할 **유성 개수**를 지정한다.\
    -   고정값이어도 되고, 화면 크기/밀도에 따라 **동적 계산**도
        가능하다.
-   `const newMeteors = [];`
    -   반복문에서 만든 유성 객체를 잠시 담아둘 **임시 배열**.
-   `for (let i = 0; i < numberOfMeteors; i++) { ... }`
    -   유성 개수만큼 반복하며 **유성 객체를 생성 →
        `newMeteors.push(...)`** 한다.
-   `setMeteors(newMeteors)`
    -   반복문이 끝난 뒤 **한 번만** 호출하여 상태를 갱신한다.

### 유성 객체 필드 의미

-   `id`: 리스트 렌더링용 **고유 키** (보통 `i` 사용).
-   `size`: 유성 **크기(px)**. 예) `Math.random() * 2 + 1` → 1\~3px의
    얇은 선/점.
-   `x`: 시작 **가로 위치(px)**. `Math.random() * window.innerWidth` 로
    화면 어디에서든 시작 가능.
-   `y`: 시작 **세로 위치(px)**. `window.innerHeight * 0.2` 처럼 **상단
    20%**에서만 시작하게 제한하면 위에서 스치는 느낌 연출이 쉬움.
-   `delay`: **애니메이션 지연(초)**. 유성들이 서로 다른 타이밍으로
    시작하도록 **어긋남**을 만든다.
-   `animationDuration`: **애니메이션 재생 시간(초)**. 유성의
    빠르기/느림을 결정한다.

### 중요 포인트

-   반복문 안에서 **매번 `setMeteors` 호출 금지** → `newMeteors`에 모아
    **한 번만** `setMeteors(newMeteors)`.
-   **좌표 단위 통일**: 모두 `px` 사용. (부모가 `position: fixed` → 자식
    `absolute` 기준이 뷰포트라 위치가 정확함)

## 2) `StarBackground`에 상태 추가

``` js
const [meteors, setMeteors] = useState([]);
```

### 역할

-   `meteors`: 화면에 **그려질 유성 목록(배열)**.
-   `setMeteors`: `generateMeteors`에서 만든 **새 배열을 반영**.

### 왜 필요한가?

-   React는 **상태 변경 시** 다시 렌더링한다.\
    유성 목록을 상태로 관리해야 `map`으로 DOM을 만들고, 재생성/갱신도
    가능하다.

### 주의

-   **철자 통일**: `meteors` / `setMeteors`\
    (`metros`, `Meteros` 등 오탈자 주의 --- 참조 에러/혼동 유발)

------------------------------------------------------------------------

## 3) `return`에 `meteors.map(...)`로 렌더링 추가

``` jsx
{meteors.map((meteor) => (
  <div
    key={`meteor-${meteor.id}`}
    className="meteor"
    style={{
      left: `${meteor.x}px`,
      top: `${meteor.y}px`,
      width: `${meteor.size}px`,
      height: `${meteor.size}px`,
      position: "absolute",
      animationDelay: `${meteor.delay}s`,
      animationDuration: `${meteor.animationDuration}s`,
    }}
  />
))}
```

### 핵심 포인트

-   **key**: `key={`meteor-\${meteor.id}`}` 처럼 **안정적인 키** 지정.
-   **스타일 연결**:
    -   `left/top`: `x/y` → `px` 단위 적용.
    -   `width/height`: `size` → `px` 단위 적용.
    -   `animationDelay`: `s` 단위 필수.
    -   `animationDuration`: `s` 단위 필수.
    -   `position: "absolute"`: 정확한 좌표 배치.
-   **클래스 연결**: `className="meteor"`
    -   CSS에서 `.meteor`의 기본 모양(선/점/그라디언트)과 **이동
        keyframes**를 정의해두면, `delay`/`duration`이 그대로 애니메이션
        타이밍에 반영된다.

### 흔한 실수

-   `meteors.mep` → **오탈자**. 반드시 `meteors.map`.
-   유성 배열에 별 속성(`opacity` 등) 혼용, 혹은 **클래스 뒤바꿈**(별을
    `.meteor`로, 유성을 `.star`로).
-   `animationDelay`/`animationDuration`에 **단위(`s`)** 빠뜨리기.

------------------------------------------------------------------------

## 흐름 요약 (실행 순서)

1.  컴포넌트 마운트 → `useEffect` 실행.\
2.  `generateStars()` & `generateMeteors()` **각 1회 호출** → 상태에
    **별/유성 목록 저장**.\
3.  `return`에서 `stars.map(...)`, `meteors.map(...)`로 **별/유성 DOM
    생성**.\
4.  CSS의 `.star`, `.meteor` + **keyframes**가 스타일/움직임 적용.
    -   `delay`/`duration`으로 유성마다 **시작 타이밍**과 **속도**가
        다르게 보임.

------------------------------------------------------------------------

## 네가 적어둔 용어/순서와 정확 매칭 요약

-   **`const generateMeteors` 생성**
    -   `const numberOfMeteors` 선언 → 만들 유성 개수\
    -   `const newMeteors = []` 선언 → 유성 임시 저장\
    -   **for 반복문**: `i = 0` → `numberOfMeteors` 미만, 유성 객체
        `newMeteors.push(...)`\
    -   마지막에 **`setMeteors(newMeteors)`**로 한 번에 반영
-   **`export const StarBackground` 내부 상태 추가**
    -   `const [meteors, setMeteors] = useState([])`\
        → 유성 목록 상태/업데이터 생성
-   **`return`에 `meteors.map(...)` 추가**
    -   `meteors.map((meteor) => <div ... />)`\
    -   `key`,
        `style(left/top/size/animationDelay/animationDuration/position)`
        지정\
    -   `className="meteor"`로 CSS 애니메이션과 연결
