# 📘 ProjectSection 구현 기록

## 📌 사용한 TailwindCSS 클래스 표

| 대상 요소 | 클래스 | 의미/효과 |
|---|---|---|
| `<section>` | `py-24 px-4 relative` | 위아래 패딩 24·좌우 패딩 4, 상대 위치 기준 |
| 컨테이너 | `container mx-auto max-w-5xl` | 중앙 정렬, 최대 너비 5xl |
| 제목 `<h2>` | `text-3xl md:text-4xl font-bold mb-4 text-center` | 반응형 폰트 크기, 굵게, 하단 여백, 중앙 정렬 |
| 제목 강조 `<span>` | `text-primary` | 테마 프라이머리 색상 적용 |
| 서브 문구 `<p>` | `text-center text-muted-foreground mb-12 max-w-2xl mx-auto` | 중앙 정렬, 보조 텍스트 색, 최대 너비 2xl, 가운데 배치 *(※ 코드상 `max-x-2xl`는 오타 → `max-w-2xl`)* |
| 그리드 래퍼 | `grid grid-cols-1 md:grid-cols-2 lg:grid-cols-3 gap-8` | 1→2→3열 반응형 그리드, 카드 간격 8 |
| 카드 래퍼 | `group bg-card rounded-lg overflow-hidden shadow-xs card-hover` | 그룹 호버 컨텍스트, 카드 배경, 모서리 둥글게, 넘침 숨김, 미세 그림자, 커스텀 호버 |
| 이미지 컨테이너 | `h-48 overflow-hidden` | 고정 높이 12rem, 넘침 숨김 |
| 썸네일 `<img>` | `w-full h-full object-cover transition-transform duration-500 group-hover:scale-110` | 꽉 채우기, 비율 유지 잘림, 변환 트랜지션 0.5s, 그룹 호버 시 확대 |

> ✅ **발견한 오타**  
> - `max-x-2xl` → `max-w-2xl`  
> - `overflow-gidden` → `overflow-hidden`  
> - 주석의 폴더명 `porject01` → `project01`

## 🧩 코드 설명

### 1) 프로젝트 데이터 구조
```js
const projects = [{
  id: 1,
  title: "아직 미정",
  description: "곧 추가할 예정임",
  image: "/",              // public 기준 상대 경로 권장: "/projects/project01/cover.jpg"
  tags: ["", "", ""],      // 사용 기술 태그 (예: ["Next.js","Tailwind","Vercel"])
  demoUrl: "#",            // 배포 링크
  githubUrl: "#",          // 저장소 링크
}];
```
- **의도**: 카드 렌더링에 필요한 최소한의 메타데이터를 갖춘 배열.
- **팁**: `image`는 `public` 폴더 기준 절대 경로(`/projects/...`)로 두면 빌드 시 번들 없이 정적 제공이 쉬움.

### 2) 섹션/헤더/설명
```jsx
<section id="projects" ...>
  <div className="container ...">
    <h2>Featured <span>Projects</span></h2>
    <p>설명 문구...</p>
```
- **역할**: 프로젝트 섹션 앵커/제목/섹션 설명을 배치.
- 설명 문구는 **담백한 톤**으로, 프로젝트 목록 상단에 짧은 맥락을 제공합니다.

### 3) 반응형 그리드 & 카드 렌더링
```jsx
<div className="grid grid-cols-1 md:grid-cols-2 lg:grid-cols-3 gap-8">
  {projects.map((project, key) => (
    <div key={key} className="group bg-card rounded-lg overflow-hidden shadow-xs card-hover">
      <div className="h-48 overflow-hidden">
        <img src={project.image} alt={project.title} className="w-full h-full object-cover transition-transform duration-500 group-hover:scale-110" />
      </div>
    </div>
  ))}
</div>
```
- **그리드**: 화면 너비에 따라 1→2→3열로 자동 재배치.
- **카드 호버**: `group`을 카드에 주고, 내부 이미지에 `group-hover:scale-110`을 적용해 **이미지 확대** 효과를 구현.

### 4) 이미지 처리
- `object-cover`로 카드 비율과 상관없이 이미지 **중앙 기준 채우기**.
- 컨테이너 `overflow-hidden`으로 확대 시 밖으로 튀어나가지 않게 처리.

## 🔧 개선/보완 포인트 (메모)

1. **오타 수정**
   - `max-x-2xl` → `max-w-2xl`
   - `overflow-gidden` → `overflow-hidden`
   - `porject01` → `project01`

2. **접근성/시맨틱**
   - `alt={project.title}`는 좋음. 설명이 필요하면 `alt={`${project.title} 썸네일`}`처럼 구체화.
   - 카드 전체를 `<a>`로 감싸거나, 버튼 2개(데모/깃허브)를 **명확한 초점 가능 요소**로 제공.

3. **링크 버튼 (선택)**
   ```jsx
   <div className="flex items-center gap-2 p-4">
     <a href={project.demoUrl} target="_blank" rel="noopener noreferrer" className="text-sm px-3 py-1 rounded-md bg-primary text-primary-foreground">Demo</a>
     <a href={project.githubUrl} target="_blank" rel="noopener noreferrer" className="text-sm px-3 py-1 rounded-md bg-secondary">GitHub</a>
   </div>
   ```

4. **태그 표시 (선택)**
   ```jsx
   <div className="flex flex-wrap gap-2 p-4">
     {project.tags?.filter(Boolean).map((tag, i) => (
       <span key={i} className="text-xs px-2 py-1 rounded-full bg-secondary/70">{tag}</span>
     ))}
   </div>
   ```

5. **이미지 경로 권장**
   - `public/projects/project01/cover.jpg` → `image: "/projects/project01/cover.jpg"`

6. **키 값**
   - `key={project.id || key}`처럼 **안정적인 키** 사용 권장.

## 🗒️ 요약(배운 점)
- **그리드/카드/이미지 호버** 조합으로 포트폴리오 프로젝트 목록을 심플하게 구성함.  
- `group` + `group-hover`로 **부모-자식 호버 인터랙션**을 쉽게 구현.  
- `public` 절대 경로를 사용해 정적 이미지 제공을 단순화.  
- 작은 오타(`max-w`, `overflow-hidden`)가 레이아웃/스타일에 영향을 줄 수 있으므로 **클래스명 검수 습관화**.
